# docs — MkDocs Documentation Site

Built with MkDocs Material theme and **mkdocs-static-i18n**: English (`en`) is the default site at `/`; Brazilian Portuguese (`pt`) mirrors every `nav` page under `/pt/` as sibling `*.pt.md` files. Source lives here; deployed to GitHub Pages by `.github/workflows/docs.yml`.

**Local preview:**
```bash
uv sync --group docs --extra cli --extra tui --extra demo
uv run mkdocs serve
```

**Contributor hooks:**
```bash
uv sync --group docs --group dev --extra cli --extra tui --extra demo
uv run pre-commit install --hook-type pre-commit --hook-type pre-push
uv run pre-commit run --all-files
```

When updating TUI documentation, keep the theme contract explicit: all Textual components and their subcomponents must follow the active theme, and runtime styling must never be hardcoded outside `netbox_tui/themes/*.json`.
Also keep the composition contract explicit: contributor guidance should treat Textual widgets as React-style composable building blocks, with composition preferred over inheritance for view structure.

**Deploy (CI only):**
```bash
mkdocs gh-deploy --force --clean --verbose
```

## hooks.py

MkDocs build hook loaded via `mkdocs.yml`:
```yaml
hooks:
  - docs/hooks.py
```

Called during the MkDocs build phase (`on_pre_build` and `on_post_page`). `on_pre_build` reads `docs/generated/raw/*.json` and generates separate reference trees under `docs/reference/cli/command-examples/` and `docs/reference/tui/launch-examples/`, **including parallel `*.pt.md` pages** (translated tab labels and prose; same command output). `on_post_page` rewrites `<img src="/assets/...">` to a path relative to each output HTML file so images resolve on GitHub Pages: the default locale writes under `site/`, while `pt` writes under `site/pt/` but static files stay in `site/assets/`. TUI screenshot gallery pages intentionally use root-absolute `/assets/screenshots/...` in Markdown so `mkdocs build --strict` does not fail on translated `*.pt.md` (doc-relative `../assets/` is mis-validated for suffix translations). See `hooks.py` for the exact behavior.

## generated/

**Never edit files in `docs/generated/` by hand.** They are overwritten by:

```bash
nbx docs generate-capture
```

This command runs `netbox_cli/docgen_capture.py`, which:
1. Loads the trimmed CLI/TUI capture spec inventory from `netbox_cli/docgen_specs.py`
2. Invokes the current checkout's CLI in isolated subprocesses
3. Writes captured output as `nbx-command-capture.md` and `nbx-command-capture.pt.md`
4. Writes raw JSON per-command to `raw/*.json`

In CI (`docs.yml`), docgen runs against `demo.netbox.dev` using `DEMO_USERNAME` / `DEMO_PASSWORD` secrets. If secrets are absent, the step is skipped and existing generated files are used.

## Navigation (from mkdocs.yml)

```
Home
Getting Started → Installation, Configuration, Quick Start
CLI Reference   → Commands, Dynamic Commands, Demo Profile
SDK             → Authentication, Facade API, Typed API, Making Requests, Schema Indexing, Error Handling
TUI             → Themes, Keyboard Shortcuts
CLI Captures    → index, Core Commands, Schema Discovery, GraphQL and HTTP,
                  Dynamic Commands, Developer Tools, Demo Profile
TUI Captures    → index, Main Browser, Logs Viewer, Developer Workbench,
                  CLI Builder, Django Models Browser
Developer Guide → Architecture, Textual Composition Pattern, Documentation Generation
```

To add a new page: create the English `.md` file, add a matching `*.pt.md` Portuguese mirror, add an entry to the `nav:` section in `mkdocs.yml`, and add the English nav label to `nav_translations` under the `pt` language block in the `i18n` plugin config. `tests/test_docs_alignment.py` asserts every `nav` Markdown path has a `*.pt.md` sibling.

Keep repo metadata aligned with the docs:
- `mkdocs.yml` branding and repo links should point to `netbox-sdk`
- `pyproject.toml` `project.urls.Documentation` should match the site URL
- SDK docs must describe all three supported access layers: raw client, facade, and typed client
